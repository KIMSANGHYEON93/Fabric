## Microsoft Fabric의 Dataflows Gen2 이해

100 경험치

이 시나리오에서는 데이터를 표준화하고 비즈니스에 대한 액세스를 제공할 수 있는 시맨틱 모델을 개발해야 합니다. Dataflows Gen2를 사용하면 다양한 데이터 소스에 연결하여 데이터를 준비하고 변환할 수 있습니다. 데이터를 레이크하우스에 직접 저장하거나 다른 목적지로 데이터 파이프라인을 사용할 수 있습니다.

## 데이터 흐름이란 무엇인가요?

*데이터 흐름은* 확장 가능한 데이터 변환 프로세스를 구축하고 실행하기 위한 클라우드 기반 ETL( *추출, 변환, 로드 ) 도구의 한 유형입니다.*

데이터 흐름 Gen2를 사용하면 다양한 소스에서 데이터를 추출하고, 다양한 변환 작업을 통해 데이터를 변환하고, 대상에 로드할 수 있습니다. Power Query Online을 사용하면 이러한 작업을 수행할 수 있는 시각적 인터페이스도 제공됩니다.

기본적으로 데이터 흐름에는 데이터 준비 시간을 줄이기 위한 모든 변환이 포함되며, 이를 새 테이블에 로드하거나 데이터 파이프라인에 포함하거나 데이터 분석가가 데이터 소스로 사용할 수 있습니다.

## Dataflows Gen2 사용 방법

전통적으로 데이터 엔지니어는 다운스트림 분석을 위해 데이터를 추출, 변환하고 사용 가능한 형식으로 로드하는 데 상당한 시간을 소비합니다. Dataflows Gen2의 목표는 Power Query Online을 사용하여 ETL 작업을 수행하는 쉽고 재사용 가능한 방법을 제공하는 것입니다.

데이터 파이프라인만 사용하기로 선택한 경우, 데이터를 복사한 후 원하는 코딩 언어를 사용하여 데이터를 추출, 변환 및 로드합니다. 또는 먼저 Dataflow Gen2를 생성하여 데이터를 추출하고 변환할 수 있습니다. 레이크하우스 및 기타 목적지에 데이터를 로드할 수도 있습니다. 이제 기업은 큐레이션된 시맨틱 모델을 쉽게 활용할 수 있습니다.

데이터 흐름에 데이터 대상을 추가하는 것은 선택 사항이며, 데이터 흐름은 모든 변환 단계를 유지합니다. 변환 후 다른 작업을 수행하거나 다른 대상에 데이터를 로드하려면 데이터 파이프라인을 생성하고 Dataflow Gen2 활동을 오케스트레이션에 추가하세요.

*또 다른 옵션은 ELT (추출, 로드, 변환)* 프로세스 에 데이터 파이프라인과 Dataflow Gen2를 사용하는 것입니다 . 이 경우 파이프라인을 사용하여 데이터를 추출하고 레이크하우스와 같은 원하는 대상에 로드합니다. 그런 다음 레이크하우스 데이터에 연결하여 데이터를 정제하고 변환하는 Dataflow Gen2를 생성합니다. 이 경우, 데이터 분석가가 보고서를 개발할 수 있도록 Dataflow를 큐레이션된 의미 모델로 제공합니다.

데이터 흐름은 수평적으로 분할될 수도 있습니다. 글로벌 데이터 흐름을 생성하면 데이터 분석가는 데이터 흐름을 사용하여 특정 요구 사항에 맞는 특수 의미 모델을 생성할 수 있습니다.

데이터 흐름을 사용하면 재사용 가능한 ETL 로직을 활용하여 데이터 소스에 대한 연결을 추가로 생성할 필요가 없습니다. 데이터 흐름은 다양한 변환 기능을 제공하며, 수동으로 실행하거나, 새로 고침 일정에 따라 실행하거나, 데이터 파이프라인 오케스트레이션의 일부로 실행할 수 있습니다.

## 이점과 제한 사항

Microsoft Fabric에서 데이터를 ETL 또는 ELT하는 방법은 여러 가지가 있습니다. Dataflows Gen2를 사용할 때의 이점과 한계를 고려해 보세요.

이익:

- 표준 날짜 차원 테이블과 같은 일관된 데이터로 데이터를 확장합니다.
- 셀프 서비스 사용자가 데이터 웨어하우스의 하위 집합에 개별적으로 액세스할 수 있도록 허용합니다.
- 재사용을 위해 데이터를 한 번 추출할 수 있는 데이터 흐름을 통해 성능을 최적화하고, 느린 소스의 데이터 새로 고침 시간을 줄입니다.
- 대규모 분석가 그룹에만 데이터 흐름을 노출하여 데이터 소스의 복잡성을 단순화합니다.
- 사용자가 데이터를 대상에 로드하기 전에 데이터를 정리하고 변환할 수 있도록 하여 데이터의 일관성과 품질을 보장합니다.
- 다양한 소스에서 데이터를 수집하는 로우코드 인터페이스를 제공하여 데이터 통합을 간소화합니다.

제한 사항:

- 데이터 흐름은 데이터 웨어하우스를 대체하지 않습니다.
- 행 수준 보안은 지원되지 않습니다.
- 패브릭 용량 작업 공간이 필요합니다.
