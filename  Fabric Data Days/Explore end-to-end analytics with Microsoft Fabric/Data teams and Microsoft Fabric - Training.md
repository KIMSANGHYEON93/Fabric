## 데이터 팀과 Microsoft Fabric 살펴보기

100 경험치

Microsoft Fabric의 통합 데이터 분석 플랫폼은 데이터 전문가가 프로젝트에서 더욱 쉽게 협업할 수 있도록 지원합니다. Fabric은 데이터 사일로와 여러 시스템의 필요성을 제거하여 데이터 전문가 간의 협업을 향상시킵니다.

## 전통적인 역할과 과제

기존의 분석 개발 프로세스에서는 데이터 팀이 데이터 작업과 워크플로우를 구분하는 데 어려움을 겪는 경우가 많습니다.

데이터 엔지니어는 분석가를 위해 데이터를 처리하고 큐레이션하며, 분석가는 이를 활용하여 비즈니스 보고서를 작성합니다. 이 과정에는 광범위한 조율이 필요하며, 이로 인해 종종 지연과 오해가 발생합니다.

데이터 분석가는 Power BI 보고서를 생성하기 전에 다운스트림 데이터 변환을 수행해야 하는 경우가 많습니다. 이 프로세스는 시간이 많이 소요되고 필요한 맥락 정보가 부족하여 분석가가 데이터와 직접 연결하기 어렵게 만듭니다.

데이터 과학자들은 네이티브 데이터 과학 기술을 기존 시스템에 통합하는 데 어려움을 겪습니다. 기존 시스템은 종종 복잡하고, 이로 인해 데이터 기반 통찰력을 효율적으로 제공하기가 어렵습니다.

## 협업 워크플로의 진화

Microsoft Fabric은 도구를 SaaS 플랫폼으로 통합하여 분석 개발 프로세스를 간소화합니다. Fabric을 사용하면 여러 역할의 담당자가 중복 작업 없이 효과적으로 협업할 수 있습니다.

**데이터 엔지니어는** 워크플로를 자동화하고 스케줄링을 지원하는 파이프라인을 사용하여 OneLake에 직접 데이터를 수집, 변환 및 로드할 수 있습니다. 효율적인 저장 및 버전 관리를 위해 Delta-Parquet 형식을 사용하여 레이크하우스에 데이터를 저장할 수 있습니다. 노트북은 복잡한 변환을 위한 고급 스크립팅 기능을 제공합니다.

**분석 엔지니어는** 레이크하우스에서 데이터 자산을 큐레이션하고, 데이터 품질을 보장하며, 셀프서비스 분석을 지원함으로써 데이터 엔지니어링과 분석 간의 격차를 해소합니다. 또한 Power BI에서 시맨틱 모델을 생성하여 데이터를 효과적으로 구성하고 표현할 수 있습니다.

**데이터 분석가는** 데이터 흐름을 사용하여 업스트림 데이터를 변환하고, Direct Lake 모드를 통해 OneLake에 직접 연결하여 다운스트림 변환의 필요성을 줄일 수 있습니다. 또한 Power BI를 사용하여 대화형 보고서를 더욱 효율적으로 만들 수 있습니다.

**데이터 과학자는** Python과 Spark를 지원하는 통합 노트북을 사용하여 머신러닝 모델을 구축하고 테스트할 수 있습니다. 레이크하우스에 데이터를 저장하고 액세스하며, Azure Machine Learning과 통합하여 모델을 운영하고 배포할 수 있습니다.

**로우코드 또는 노코드 사용자** 와 **시민 개발자는** OneLake 카탈로그를 통해 큐레이션된 데이터 세트를 검색하고 Power BI 템플릿을 사용하여 보고서와 대시보드를 빠르게 만들 수 있습니다. 또한 데이터 엔지니어의 도움 없이 데이터 흐름을 사용하여 간단한 ETL 작업을 수행할 수 있습니다.
