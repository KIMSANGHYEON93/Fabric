## 이벤트스트림 변환

100 경험치

원시 스트리밍 데이터는 분석이나 작업에 필요한 정확한 형식으로 수신되는 경우가 드뭅니다. 변환 기능을 사용하면 데이터를 목적지로 라우팅하기 전에 데이터를 정제, 보강 및 재구성하여 각 엔드포인트가 특정 목적에 맞게 최적화된 데이터를 수신하도록 할 수 있습니다.

일반적인 변환 시나리오는 다음과 같습니다.

- **데이터 품질**: 처리 전에 유효하지 않거나 불완전한 데이터를 필터링합니다.
- **콘텐츠 기반 라우팅**: 실제 데이터 값이나 콘텐츠를 기반으로 다양한 데이터 하위 집합을 적절한 목적지로 라우팅합니다.
- **데이터 강화**: 계산된 필드 추가, 명확성을 위해 열 이름 변경 또는 다운스트림 호환성을 위해 데이터 유형 변환
- **집계 및 요약**: 대시보드 표시를 위해 시간 창에 따른 누적 합계, 평균 또는 개수를 계산합니다.
- **형식 표준화**: 스트림을 결합하기 전에 여러 데이터 소스에서 일관된 데이터 구조를 보장합니다.

## 이벤트 데이터 변환

이벤트스트림 캔버스를 사용하면 이벤트 데이터 처리 워크플로를 생성할 수 있습니다. 이벤트스트림은 캔버스로 드래그 앤 드롭할 수 있는 여러 가지 코드 없는 변환 기능을 제공합니다.

- **필터**: 입력 필드 값을 기준으로 이벤트를 필터링합니다. 특정 조건을 충족하는 이벤트만 필터링합니다. 예: 온도 > 80°, 상태 = "오류", 고객 유형 = "프리미엄".
- **필드 관리**: 이 변환을 사용하면 입력 또는 다른 변환에서 들어오는 필드를 추가, 제거, 데이터 유형 변경 또는 이름 변경이 가능합니다. 계산된 필드를 추가하고, 불필요한 열을 제거하고, 필드 이름을 변경하고, 대상 요구 사항에 맞게 데이터 유형을 변경하세요.
- **집계**: 집계 변환을 사용하여 일정 기간 동안 새 이벤트가 발생할 때마다 집계(합계, 최소값, 최대값 또는 평균)를 계산합니다. 이 작업을 통해 계산된 열의 이름을 변경하고 데이터의 다른 차원을 기준으로 집계를 필터링할 수도 있습니다. 동일한 변환에 하나 이상의 집계를 포함할 수 있습니다.
- **그룹화 기준**: 시간 단위 내 이벤트 간 집계를 계산합니다(예: 시간당 판매량 합계 또는 일별 평균 기온). 이 변환은 *텀블링 윈도우* (고정 간격) 및 *슬라이딩 윈도우* (겹치는 간격)를 포함한 다양한 시간 단위를 지원합니다.
- **Union**: Union 변환을 사용하여 이벤트 캔버스에서 두 개 이상의 노드를 연결하고, 이름과 데이터 유형이 동일한 공유 필드를 가진 이벤트를 하나의 테이블에 추가합니다. 일치하지 않는 필드는 삭제되고 출력에 포함되지 않습니다.
- **조인**: 두 스트림의 데이터를 일치 조건에 따라 결합합니다.
- **확장**: 이 배열 변환을 사용하여 배열 내의 각 값에 대해 새 행을 만듭니다.

## 변환 워크플로 생성

변환을 함께 사용하여 데이터 처리 파이프라인을 만들 수 있습니다. 예를 들어, 장비 온도 측정값 스트림이 있는 경우, **필터를** 사용하여 수신 IoT 데이터에서 센서 오류를 제거하는 것부터 시작할 수 있습니다. 다음으로, **필드 관리를** 사용하여 온도 임계값을 기반으로 계산된 "우선순위" 열을 추가할 수 있습니다. 그런 다음, **그룹화 기준으로** 위치별 시간당 평균을 계산할 수 있습니다. 마지막으로, 처리된 데이터를 적절한 목적지로 라우팅합니다. 온도 데이터는 규칙 평가를 위해 Fabric Activator로, 시간별 요약은 과거 분석을 위해 Lakehouse로 전송합니다.
