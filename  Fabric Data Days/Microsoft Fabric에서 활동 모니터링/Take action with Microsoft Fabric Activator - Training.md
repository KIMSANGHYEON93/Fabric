## Microsoft Fabric Activator로 조치를 취하세요

100 경험치

표면 변화 데이터, 이상 징후 또는 중요 이벤트를 모니터링할 때 경고가 생성되거나 작업이 트리거됩니다. 실시간 데이터 분석은 일반적으로 특정 시점 이벤트와 관련된 영구적인 일련의 데이터로 구성된 데이터 스트림의 수집 및 처리를 기반으로 합니다. 예를 들어 환경 IoT 기상 센서의 데이터 스트림이 있습니다. Fabric의 실시간 인텔리전스에는 스트리밍 데이터에 대한 작업을 트리거하는 데 사용할 수 있는 Activator라는 도구가 포함되어 있습니다. 예를 들어 환경 IoT 기상 센서의 데이터 스트림은 풍속 임계값이 충족될 때 선원에게 이메일을 발송하는 데 사용될 수 있습니다. 특정 조건이나 로직이 충족되면 사용자에게 경고하거나, 파이프라인과 같은 Fabric 작업 항목을 실행하거나, Power Automate 워크플로를 시작하는 등의 작업이 수행됩니다. 로직은 정의된 임계값, 일정 기간 동안 반복적으로 발생하는 이벤트와 같은 패턴 또는 Kusto Query Language(KQL) 쿼리로 정의된 로직의 결과일 수 있습니다.

## Activator란 무엇인가

Activator는 Microsoft Fabric의 기술로, 동작을 트리거하는 이벤트의 자동 처리를 지원합니다. 예를 들어, Activator를 사용하면 이벤트 스트림의 값이 특정 범위를 벗어날 때 이메일로 알림을 받거나, 실시간 대시보드가 업데이트될 때 Spark 기반 데이터 처리 로직을 수행하는 노트북을 실행할 수 있습니다.

![Microsoft Fabric의 Activator 알림 스크린샷입니다.](https://learn.microsoft.com/en-us/training/wwl/monitor-fabric-items/media/activator.png)

## Activator 핵심 개념 이해

*Activator는 이벤트*, 객체, *속성*, *규칙* 이라는 4가지 핵심 개념을 기반으로 작동합니다 .

- **이벤트**  - 데이터 스트림의 각 레코드는 특정 시점에 발생한 *이벤트를 나타냅니다.*
- **객체**  - 이벤트 레코드의 데이터는 판매 주문, 센서 또는 기타 비즈니스 엔터티와 같은 *객체를 나타내는 데 사용될 수 있습니다.*
- **속성** \- 이벤트 데이터의 필드는 비즈니스 객체의 *속성 에 매핑되어 해당 객체의 상태 중 일부를 나타낼 수 있습니다. 예를 들어,* *total\_amount* 필드는 판매 주문 총액을 나타내고, *temperature* 필드는 환경 센서에서 측정한 온도를 나타낼 수 있습니다.
- **규칙** \- Activator를 사용하여 이벤트 기반 작업을 자동화하는 핵심은 이벤트에서 참조되는 객체의 속성 값에 따라 작업이 트리거되는 조건을 설정하는 *규칙을* 정의하는 것입니다. 예를 들어, 센서에서 측정된 온도가 특정 임계값을 초과하면 유지 관리 관리자에게 이메일을 전송하는 규칙을 정의할 수 있습니다.

## Activator의 사용 사례

Activator는 동적 재고 관리, 실시간 고객 참여, 클라우드 환경에서의 효과적인 리소스 할당 등 다양한 상황에서 도움을 줄 수 있습니다. 실시간 데이터 분석 및 조치가 필요한 모든 상황에 강력한 도구입니다.

Activator를 사용하면 다음을 수행할 수 있습니다.

- 제품 판매가 감소하면 마케팅 활동을 시작하세요.
- 온도 변화가 부패하기 쉬운 상품에 영향을 미칠 수 있는 경우 알림을 보냅니다.
- 앱과 웹사이트에서 사용자 경험에 영향을 미치는 실시간 문제를 표시합니다.
- 예상 기간 내에 배송물이 업데이트되지 않으면 알림을 보냅니다.
- 고객 계좌 잔액이 특정 임계값을 넘으면 알림을 보냅니다.
- 데이터 처리 워크플로의 이상이나 오류에 즉시 대응합니다.
- 동일 매장 매출이 감소하면 광고를 게재하세요.
- 식료품점의 냉동고가 고장나기 전에 식료품을 옮기도록 매장 관리자에게 알리세요.
