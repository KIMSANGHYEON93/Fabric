## Microsoft Fabric 레이크하우스와 함께 작업하세요

100 경험치

이제 Microsoft Fabric 레이크하우스의 핵심 기능을 이해했으므로 이를 사용하는 방법을 살펴보겠습니다.

## 호숫가 주택을 만들고 탐험하세요

새로운 레이크하우스를 만들면 작업 공간에 세 가지 다른 데이터 항목이 자동으로 생성됩니다.

- 레이크 **하우스** 에는 바로가기, 폴더, 파일, 테이블이 포함되어 있습니다.
- **의미 모델(기본값)은** Power BI 보고서 개발자에게 쉬운 데이터 소스를 제공합니다.
- SQL **분석 엔드포인트는** SQL을 사용하여 데이터를 쿼리하는 데 대한 읽기 전용 액세스를 허용합니다.

![설명된 대로 Lakehouse 항목 3개의 스크린샷입니다.](https://learn.microsoft.com/en-us/training/wwl/get-started-lakehouses/media/lakehouse-items.png)

레이크하우스의 데이터는 두 가지 모드로 작업할 수 있습니다.

- **lakehouse를** 사용하면 lakehouse에서 테이블, 파일, 폴더를 추가하고 상호 작용할 수 있습니다.
- **SQL 분석 엔드포인트를** 사용하면 SQL을 사용하여 레이크하우스의 테이블을 쿼리하고 관계형 의미 모델을 관리할 수 있습니다.

![두 개의 레이크하우스 익스플로러 모드의 스크린샷입니다.](https://learn.microsoft.com/en-us/training/wwl/get-started-lakehouses/media/explorer-modes.png)

## 레이크하우스에 데이터 수집

레이크하우스에 데이터를 수집하는 것은 ETL 프로세스의 첫 단계입니다. 다음 방법 중 하나를 사용하여 데이터를 레이크하우스로 가져오세요.

- **업로드**: 로컬 파일을 업로드합니다.
- **데이터 흐름 Gen2**: Power Query를 사용하여 데이터를 가져오고 변환합니다.
- **노트북**: Apache Spark를 사용하여 데이터를 수집, 변환, 로드합니다.
- **데이터 팩토리 파이프라인**: 데이터 복사 활동을 사용합니다.

이 데이터는 파일이나 테이블에 직접 로드할 수 있습니다. 데이터를 수집할 때 데이터 로드 패턴을 고려하여 처리 전에 모든 원시 데이터를 파일로 로드해야 하는지, 아니면 스테이징 테이블을 사용해야 하는지 결정하십시오.

**Spark 작업 정의를** 사용하여 Spark 클러스터에 배치/스트리밍 작업을 제출할 수도 있습니다. 다양한 언어(예: Java의.jar)의 컴파일 출력에서 바이너리 파일을 업로드하여 레이크하우스에 호스팅된 데이터에 다양한 변환 로직을 적용할 수 있습니다. 바이너리 파일 외에도 더 많은 라이브러리와 명령줄 인수를 업로드하여 작업의 동작을 더욱 세부적으로 사용자 지정할 수 있습니다.

## 단축키를 사용하여 데이터 액세스

Fabric에서 데이터에 액세스하고 사용하는 또 다른 방법은 *바로가기를* 사용하는 것입니다. 바로가기를 사용하면 데이터를 외부 저장소에 저장한 채로 레이크하우스에 통합할 수 있습니다.

바로가기는 다른 스토리지 계정이나 다른 클라우드 제공업체에 있는 데이터를 가져와야 할 때 유용합니다. 레이크하우스 내에서 다른 스토리지 계정과 데이터 웨어하우스, KQL 데이터베이스, 기타 레이크하우스와 같은 다른 Fabric 항목을 가리키는 바로가기를 만들 수 있습니다.

원본 데이터 권한 및 자격 증명은 모두 OneLake에서 관리합니다. 다른 OneLake 위치로 가는 바로가기를 통해 데이터에 액세스할 때, 호출하는 사용자의 ID를 사용하여 바로가기의 대상 경로에 있는 데이터에 대한 액세스를 승인합니다. 사용자는 대상 위치에서 데이터를 읽을 수 있는 권한이 있어야 합니다.

바로가기는 레이크하우스 데이터베이스와 KQL 데이터베이스 모두에서 생성할 수 있으며, 레이크에서는 폴더 형태로 나타납니다. 이를 통해 Spark, SQL, 실시간 인텔리전스 및 Analysis Services 모두 데이터 쿼리 시 바로가기를 활용할 수 있습니다.
