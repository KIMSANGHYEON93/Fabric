## Microsoft Fabric 레이크하우스를 탐험하세요

100 경험치

레이크 **하우스는** 데이터베이스로 표현되며, 델타 형식 테이블을 사용하여 데이터 레이크 위에 구축됩니다. 레이크하우스는 관계형 데이터 웨어하우스의 SQL 기반 분석 기능과 데이터 레이크의 유연성 및 확장성을 결합합니다. 레이크하우스는 모든 데이터 형식을 저장하며 다양한 분석 도구 및 프로그래밍 언어와 함께 사용할 수 있습니다. 클라우드 기반 솔루션인 레이크하우스는 자동으로 확장되고 고가용성 및 재해 복구 기능을 제공합니다.

![데이터 레이크의 폴더 구조와 데이터 웨어하우스의 관계적 기능을 보여주는 레이크하우스 다이어그램입니다.](https://learn.microsoft.com/en-us/training/wwl/get-started-lakehouses/media/lakehouse-components.png)

호숫가 주택의 장점은 다음과 같습니다.

- 레이크하우스는 Spark 및 SQL 엔진을 사용하여 대규모 데이터를 처리하고 머신 러닝이나 예측 모델링 분석을 지원합니다.
- *Lakehouse 데이터는 스키마 온 리드(schema-on-read) 형식* 으로 구성됩니다 . 즉, 미리 정의된 스키마가 아니라 필요에 따라 스키마를 정의할 수 있습니다.
- 레이크하우스는 데이터 일관성과 무결성을 위해 Delta Lake 포맷 테이블을 통해 ACID(원자성, 일관성, 고립성, 내구성) 트랜잭션을 지원합니다.
- 레이크하우스는 데이터 엔지니어, 데이터 과학자, 데이터 분석가가 데이터에 접근하고 사용할 수 있는 단일 장소입니다.

데이터 일관성을 유지하면서 확장 가능한 분석 솔루션을 원한다면 레이크하우스가 좋은 선택입니다. 어떤 솔루션이 가장 적합한지 결정하려면 구체적인 요구 사항을 평가하는 것이 중요합니다.

## 레이크하우스에 데이터 로드

패브릭 레이크하우스는 분석 솔루션의 핵심 요소입니다. ETL(추출, 변환, 로드) 프로세스에 따라 레이크하우스에 데이터를 로드하기 전에 데이터를 수집하고 변환할 수 있습니다.

로컬 파일, 데이터베이스 또는 API를 포함한 다양한 소스에서 여러 가지 일반적인 형식으로 데이터를 수집할 수 있습니다. Azure Data Lake Store Gen2 또는 OneLake와 같은 외부 소스의 데이터에 대한 Fabric **바로 가기를** 만들 수도 있습니다. Lakehouse 탐색기를 사용하여 Fabric 플랫폼 내에서 파일, 폴더, 바로 가기 및 테이블을 탐색하고 해당 내용을 확인하세요.

수집된 데이터는 Apache Spark와 Notebooks 또는 Dataflows Gen2를 사용하여 변환한 후 로드할 수 있습니다. Data Factory 파이프라인을 사용하여 다양한 ETL 작업을 조율하고 준비된 데이터를 레이크하우스에 저장하세요.

다음을 포함한 여러 가지 이유로 호숫가 주택을 사용할 수 있습니다.

- SQL을 사용하여 분석합니다.
- 머신 러닝 모델을 훈련합니다.
- 실시간 데이터에 대한 분석을 수행합니다.
- Power BI에서 보고서를 개발합니다.

## 호숫가 주택을 확보하세요

레이크하우스 액세스는 작업 공간 또는 항목 수준 공유를 통해 관리됩니다. 작업 공간 역할은 작업 공간 내의 모든 항목에 대한 액세스 권한을 부여하므로 공동 작업자에게 사용해야 합니다. 항목 수준 공유는 분석이나 Power BI 보고서 개발과 같이 읽기 전용 요구 사항에 대한 액세스 권한을 부여하는 데 가장 적합합니다.

Fabric 레이크하우스는 민감도 레이블을 포함한 데이터 거버넌스 기능도 지원하며, Fabric 테넌트와 함께 Microsoft Purview를 사용하여 확장할 수 있습니다.
