## Fabric API를 사용하여 CI/CD 자동화

완전한 100 경험치

- 2분

Fabric REST API를 사용하면 Fabric 프로시저와 프로세스를 자동화하여 효율성과 생산성을 향상시킬 수 있습니다. REST API는 Representational State Transfer Programming Application Interface의 약자입니다. Azure REST API는 다양한 Azure 서비스를 관리하고 상호 작용하는 데 사용됩니다.

Fabric REST API를 사용하면 다음과 같은 이점이 있습니다.

- 일관성 있게 반복되는 프로세스를 자동화하여 지속적으로 데이터 처리를 수행하기가 더 쉬워졌습니다.
- 다른 시스템 및 애플리케이션과 원활하게 통합되어 간소화되고 효율적인 데이터 파이프라인을 제공합니다.

Fabric CI/CD REST API는 배포 파이프라인과 Git 통합에 사용할 수 있습니다.

- [배포 파이프라인 REST API](https://learn.microsoft.com/en-us/rest/api/fabric/core/deployment-pipelines)
- [Git REST API](https://learn.microsoft.com/en-us/rest/api/fabric/core/git)

## CI/CD를 위한 Fabric REST API를 사용하여 프로세스 자동화

Fabric CI/CD REST API를 사용하여 다음을 수행할 수 있습니다.

- 작업 공간에서 변경한 내용을 연결된 원격 브랜치에 커밋합니다.
- 연결된 브랜치에 푸시된 커밋으로 작업 공간을 업데이트합니다.
- Git 상태 API를 사용하여 어떤 항목에 변경 사항이 들어오고 어떤 항목에 아직 Git에 커밋되지 않은 변경 사항이 있는지 확인하세요.
- 배포 파이프라인 단계 항목 나열: 지정된 배포 파이프라인의 지정된 단계에 할당된 작업 공간에서 지원되는 항목을 반환합니다.
- 배포 단계 콘텐츠: 지정된 배포 파이프라인의 지정된 단계에서 항목을 배포합니다.
