1.

Fabric의 CI/CD 프로세스에서 Git의 역할은 무엇입니까?

Git은 다양한 환경에 코드 변경 사항을 배포하는 데 사용됩니다.

Git은 코드 변경 사항을 자동으로 테스트하는 데 사용됩니다.

Git을 사용하면 팀이 브랜치를 사용하여 협업하고 버전 관리 기능을 활용할 수 있습니다. 증분 코드 변경 사항을 관리하고 코드 이력을 확인하는 데 도움이 됩니다.

2.

Fabric 작업 공간을 Git 저장소에 연결하는 목적은 무엇입니까?

작업 공간에서 변경한 내용을 라이브 환경에 자동으로 배포합니다.

작업 공간과 Git 간의 콘텐츠를 동기화하여 동일한 콘텐츠를 보유하도록 합니다.

여러 개발자가 충돌 없이 동시에 동일한 코드에서 작업할 수 있도록 합니다.

3.

Fabric에서 배포 파이프라인의 주요 기능은 무엇입니까?

배포 파이프라인은 Fabric에서 새로운 콘텐츠를 만드는 데 사용됩니다.

배포 파이프라인은 Fabric에서 오래된 콘텐츠를 삭제하는 데 사용됩니다.

배포 파이프라인은 개발, 테스트, 프로덕션 단계를 거쳐 콘텐츠가 이동하는 과정을 자동화합니다.
