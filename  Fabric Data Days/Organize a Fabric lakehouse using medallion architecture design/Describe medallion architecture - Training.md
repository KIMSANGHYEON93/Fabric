## 메달리온 아키텍처를 설명하세요

100 경험치

Fabric의 데이터 레이크하우스는 ACID(원자성, 일관성, 고립성, 내구성) 트랜잭션을 기본적으로 지원하는 델타 레이크 형식을 기반으로 구축됩니다. 이 프레임워크 내에서 *메달리온 아키텍처는* 레이크하우스의 데이터를 논리적으로 구성하는 데 사용되는 권장 데이터 설계 패턴입니다. 메달리온 아키텍처는 여러 계층을 거치면서 데이터 품질을 향상시키는 것을 목표로 합니다. 이 아키텍처는 일반적으로 브론즈(원시), 실버(검증), 골드(보강)의 세 계층으로 구성되며, 각 계층은 더 높은 데이터 품질 수준을 나타냅니다. 필요에 따라 데이터가 계층 간에 이동할 수 있다는 의미로 "멀티홉" 아키텍처라고도 합니다.

이 아키텍처는 다양한 검사 및 변경 과정을 거치면서도 데이터의 신뢰성과 일관성을 보장합니다. 또한, 데이터를 더 쉽고 빠르게 분석할 수 있도록 안전하게 저장합니다.

메달리온 아키텍처는 다른 데이터 구성 방법을 대체하는 것이 아니라 보완합니다. 메달리온 아키텍처는 데이터 아키텍처나 모델이라기보다는 데이터 정제를 위한 프레임워크라고 볼 수 있습니다. 메달리온 아키텍처는 기업이 기존 데이터 모델과 함께 메달리온 아키텍처의 이점을 활용할 수 있도록 호환성과 유연성을 보장하며, 끊임없이 변화하는 데이터 환경에서 적응력을 유지하면서 데이터 솔루션을 맞춤 설정하고 전문 지식을 유지할 수 있도록 지원합니다.

## 메달리온 아키텍처 형식을 이해하세요

### 청동층

메달리온 건축의 청동 또는 원목 층은 레이크하우스의 첫 번째 층입니다. 이곳은 정형, 반정형, 비정형 등 모든 데이터가 모이는 곳입니다. 데이터는 원래 형식 그대로 저장되며 어떠한 변경도 가해지지 않습니다.

### 은층

실버 또는 검증된 레이어는 레이크하우스의 두 번째 레이어입니다. 여기에서 데이터를 검증하고 정제합니다. 실버 레이어의 일반적인 작업에는 데이터 결합 및 병합, 그리고 Null 제거 및 중복 제거와 같은 데이터 검증 규칙 적용이 포함됩니다. 실버 레이어는 조직이나 팀 전체의 중앙 저장소로 생각할 수 있으며, 데이터가 일관된 형식으로 저장되고 여러 팀에서 액세스할 수 있습니다. 실버 레이어에서는 모든 데이터를 한곳에 정리하여 골드 레이어에서 정제 및 모델링할 수 있도록 합니다.

### 금층

골드 또는 강화된 계층은 레이크하우스의 세 번째 계층입니다. 골드 계층에서는 데이터가 특정 비즈니스 및 분석 요구에 맞게 더욱 세분화됩니다. 여기에는 일별 또는 시간별과 같이 특정 단위까지 데이터를 집계하거나 외부 정보를 활용하여 강화하는 작업이 포함될 수 있습니다. 데이터가 골드 단계에 도달하면 분석, 데이터 과학 또는 MLOps를 포함한 다운스트림 팀에서 사용할 수 있게 됩니다.

### 메달리온 아키텍처를 사용자 지정하세요

조직의 특정 사용 사례에 따라 더 많은 계층이 필요할 수 있습니다. 예를 들어, 브론즈 계층으로 변환되기 전에 특정 형식의 데이터를 가져오기 위한 추가 "원시" 계층이 있을 수 있습니다. 또는 특정 사용 사례에 맞게 더욱 정제되고 강화된 데이터를 위한 "플래티넘" 계층이 있을 수도 있습니다. 메달리온 아키텍처는 계층의 이름과 개수에 관계없이 유연하며 조직의 특정 요구 사항에 맞게 조정할 수 있습니다.

## Fabric에서 계층 간 데이터 이동

메달리온 계층 간에 데이터를 이동하면 다운스트림 데이터 활동을 위해 데이터를 정제, 구성 및 준비할 수 있습니다. Fabric의 레이크하우스에는 계층 간에 데이터를 이동하는 여러 가지 방법이 있으므로 팀에 적합한 방법을 선택할 수 있습니다.

계층 간에 데이터를 이동하고 변환하는 방법을 결정할 때 고려해야 할 몇 가지 사항이 있습니다.

- 얼마나 많은 데이터를 다루고 있나요?
- 당신이 해야 할 변환은 얼마나 복잡합니까?
- 얼마나 자주 계층 간에 데이터를 이동해야 합니까?
- 어떤 도구를 가장 편하게 사용하시나요?

데이터 변환과 데이터 오케스트레이션의 차이점을 이해하면 Fabric 내에서 작업에 적합한 도구를 선택하는 데 도움이 됩니다.

*데이터 변환은* 특정 요구 사항을 충족하도록 데이터의 구조나 내용을 변경하는 것을 포함합니다. Fabric의 데이터 변환 도구에는 데이터플로우(Gen2)와 노트북이 있습니다. 데이터플로우는 소규모 시맨틱 모델과 간단한 변환에 적합합니다. 노트북은 대규모 시맨틱 모델과 더 복잡한 변환에 더 적합합니다. 또한 노트북을 사용하면 변환된 데이터를 레이크하우스에 관리형 델타 테이블로 저장하여 보고에 활용할 수 있습니다.

*데이터 오케스트레이션은* 여러 데이터 관련 프로세스를 조정하고 관리하여 원하는 결과를 달성하기 위해 함께 작동하도록 하는 것을 의미합니다. Fabric에서 데이터 오케스트레이션을 위한 주요 도구는 *파이프라인* 입니다. 파이프라인은 데이터를 한 곳에서 다른 곳으로, 즉 메달리온 아키텍처의 한 계층에서 다음 계층으로 이동하는 일련의 단계입니다. 파이프라인은 일정에 따라 실행되도록 자동화하거나 이벤트에 의해 트리거될 수 있습니다.
