## Fabric에서 메달리온 아키텍처 구현

100 경험치

이제 메달리온 아키텍처에 대한 확실한 이해를 얻었으므로 Fabric에서 이를 구현하는 방법을 살펴보겠습니다.

- **기반 구축**: [Fabric 레이크하우스를 만드세요](https://learn.microsoft.com/en-us/fabric/onelake/create-lakehouse-onelake). 동일한 레이크하우스를 여러 메달리온 아키텍처에 사용할 수도 있고, 사용 사례에 따라 서로 다른 레이크하우스를 사용하거나, 서로 다른 작업 공간에 서로 다른 레이크하우스를 사용할 수도 있습니다.
- **아키텍처 설계**: 아키텍처 레이아웃을 만들고, 레이어를 정의하고, 레이어 간 데이터 흐름을 결정합니다. 가장 간단한 구현 방법은 Bronze를 원시 레이어로, Silver를 큐레이션 레이어로, Gold를 프레젠테이션 레이어로 사용하는 것입니다. Gold 레이어는 스타 스키마로 모델링하고 보고에 최적화해야 합니다.
	| 질문 | 청동 | 은 | 금 |
	| --- | --- | --- | --- |
	| 그 층에서는 무슨 일이 일어나는가? | 원시 데이터 수집 | 데이터 정리 및 검증 | 추가 변환 및 모델링 |
	| 어떤 도구를 사용하나요? | 파이프라인, 데이터 흐름, 노트북 | 데이터 흐름 또는 노트북 | SQL 분석 엔드포인트 또는 의미 모델 |
- **브론즈에 데이터 수집**: 브론즈 레이어에 데이터를 수집하는 방법을 결정합니다. 파이프라인, 데이터 흐름 또는 노트북을 사용하여 이 작업을 수행할 수 있습니다.
- **데이터 변환 및 실버로 로드**: 실버 계층에서 데이터를 어떻게 변환할지 결정합니다. 데이터 흐름이나 노트북을 사용하여 이 작업을 수행할 수 있습니다. 실버 수준에서의 변환은 데이터 모델링이 아닌 데이터 품질과 일관성에 중점을 두어야 합니다.
- **골드 레이어 생성**: 골드 레이어를 어떻게 생성할지, 무엇을 포함할지, 어떻게 사용할지 결정합니다.
	- 골드 레이어는 차원 모델을 사용하여 보고를 위한 데이터를 모델링하는 곳입니다. 여기서 관계를 설정하고, 측정값을 정의하고, 효과적인 보고에 필수적인 기타 요소를 통합합니다.
	- 다양한 대상 그룹이나 도메인에 대해 여러 개의 골드 레이어를 만들 수 있습니다. 예를 들어, 재무팀을 위한 골드 레이어와 영업팀을 위한 골드 레이어를 별도로 만들 수 있습니다. 또한, 머신 러닝에 최적화된 데이터 과학자를 위한 골드 레이어를 만들 수도 있습니다.
	- 필요에 따라 데이터 웨어하우스를 골드 레이어로 사용할 수도 있습니다.
	- Fabric에서는 데이터 흐름이나 노트북을 사용하여 데이터를 변환한 다음, 레이크하우스의 골드 델타 테이블에 로드할 수 있습니다. 그런 다음 SQL 분석 엔드포인트를 사용하여 델타 테이블에 연결하고 SQL을 사용하여 보고용 데이터를 모델링할 수 있습니다. 또는 Power BI를 사용하여 골드 레이어의 SQL 분석 엔드포인트에 연결하고 보고용 데이터를 모델링할 수 있습니다.
- **다운스트림 사용 활성화**: 데이터의 다운스트림 사용을 활성화하는 방법을 결정합니다. 작업 영역 또는 항목 권한을 사용하거나 SQL 분석 엔드포인트에 연결하여 이 작업을 수행할 수 있습니다.
